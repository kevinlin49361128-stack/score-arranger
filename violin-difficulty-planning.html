小提琴「加高難度」功能 — 規劃決策

使用者覺得改編出來的小提琴譜「太簡單、不夠炫技」時,讓 app 能主動加料。每一題挑一個(標「可複選」的可挑多個)。

選完按右下角「複製決策摘要」,把內容貼回給我,我就照這個一次做完。標 🎯 建議 的是我的推薦,但你說了算。

現況:我們剛做完的 enrich 功能只會「往低音方向疊和弦」(吉他思路 —— 補低音、補內聲部)。 小提琴要的是反方向 —— 八度疊高、雙音、移高把位、走句、裝飾音。

三個難度層級: Tier 1 低風險 可演奏性能機械驗證、譜面改動小; Tier 2 中風險 要新邏輯、會改變旋律輪廓; Tier 3 先跳過 人工泛音、左手撥弦、跳弓 —— 記譜與驗證都太複雜,這次不做。

1 架構起手式 — 怎麼接進現有引擎

小提琴加音的「驗證/復原/LLM 串接」要不要複用 enrich 那一套管線。

擴充 enrich 🎯 建議
讓 enrich 認得目標樂器:小提琴時改用 check_violin_chord(≤4 音、相鄰弦),新增 octave texture。
  • 復用既有驗證、undo/redo、LLM op 串接
  • 新程式碼最少,測試面也最小
  • enrich 語意被撐大,要清楚註解
新增獨立 op
為小提琴難度新開一個 op kind(例如 embellish)。
  • 語意乾淨、和 enrich 互不干擾
  • 驗證、server 分支、LLM prompt、UI describeOp 全要重做一遍
兩者分工
enrich 加樂器感知處理「疊音/雙音」;另開 op 處理「走句/裝飾」。
  • 各司其職、未來擴充清楚
  • 工作量最大,看 Tier 2 是否這輪就做

2 Tier 1 — 要納入哪些「加難度」手法 可複選

四種都是低風險、可演奏性能機械驗證。建議四種全選 —— 越多手法,LLM 越能因地制宜。

八度疊置 octave doubling 🎯 建議
把旋律音同時疊上(或下)八度,瞬間變雙音。只要兩音都在音域內就一定拉得動,最穩。
原: G4 單音  →  加料: G3+G4 八度雙音
Tier 1
雙音 / 和弦 double-stops 🎯 建議
在和聲點上,於相鄰弦補一個和聲音。經 check_violin_chord 把關(≤4 音、相鄰弦、音程可行)。
原: E5 單音(伴奏和弦為 C 大三)  →  加料: C5+E5 三度雙音
Tier 1
困難弓法 demanding bowing 🎯 建議
spiccato / staccato / martelé 等較難弓法。已能用現有 articulation op 表達 —— 納入只是讓小提琴 preset 主動建議。
既有能力: articulation op 已支援,零新引擎碼
Tier 1
移高把位 higher positions 🎯 建議
把段落整體升高,讓左手落在第 3/5/7 把位。已能用現有 transpose op 表達。
既有能力: transpose op 已支援,零新引擎碼
Tier 1

3 Tier 2 — 進階手法這輪做不做

這兩種會「改變旋律輪廓」、需要新邏輯。可以這輪就做,也可以等 Tier 1 上線穩定後再加。

只做走句化 figuration
把長音 / 持續音拆成跑動音群(分解和弦、音階)。像 enrich 的 arpeggio texture,但作用在旋律上。
原: 全音符 C5  →  加料: C5-E5-G5-E5 十六分音群
Tier 2 · 中風險
只做裝飾音 ornamentation
加顫音(trill)、漣音(mordent)、倚音(grace note)、迴音(turn)。改動小,但要處理記譜。
原: 二分音符 A4  →  加料: A4 + trill 記號
Tier 2 · 中風險
兩者都做
走句化 + 裝飾音這輪一起上。功能最完整,但這輪工作量明顯變大。
Tier 2 · 中風險
這輪都先不做 🎯 建議
先把 Tier 1 做穩、上線收使用者回饋,Tier 2 下一輪再加。範圍可控、能更快交付。

4 難度量表要不要升級

目前 difficulty.py 只算 音域 / 密度 / 和弦 / 節奏 四個因子,抓不到「技巧難度」(雙音、把位、弓法)。沒升級的話,加了難度但難度計顯示不太會動。

加 technique_factor 🎯 建議
新增第五個因子:統計雙音數、把位高度、難弓法密度。難度計才能真實反映加料效果,Phase C 的 target_difficulty 也才準。
  • 難度顯示與「加難度」功能一致,使用者有感
  • 要動 difficulty.py 與其測試
維持現狀
難度計暫不動。加料仍會被「和弦因子」抓到一部分,但雙音/把位/弓法不計入。
  • 範圍最小、最快
  • 使用者加完難度,難度計可能只微微上升

5 使用者怎麼觸發

使用者要怎麼叫出「加難度」。現有的自然語言面板(輸入「這段太簡單,加點難度」)已能承接。

只走自然語言面板 🎯 建議
使用者打「第 8–16 小節太簡單,加雙音」,LLM 產生 enrich/transpose op。零新 UI。
  • 沿用現成 NLEditDialog,最快
  • 自然語言最彈性
另開「加難度」專屬面板
獨立 UI:選聲部、選小節範圍、拉難度滑桿、勾手法。
  • 不會編曲的人也好操作、可預覽
  • 要設計新元件,工作量明顯增加
兩者都做
先上自然語言,專屬面板列為下一輪。
  • 體驗最完整
  • 這輪範圍最大

6 預設炫技強度

使用者沒特別指定「加多少」時的預設力道。三者都會經可演奏性驗證,差別在「主動加多少」。

保守 conservative
只在強拍加八度與簡單三/六度雙音。最安全、最不會「過頭」。
一句旋律約加 2–3 處雙音
平衡 balanced 🎯 建議
八度 + 雙音密度適中,偶爾建議把位移高。聽起來「有挑戰但不雜亂」。
一句旋律約加 4–6 處雙音 + 把位建議
炫技 virtuosic
盡量加滿雙音、移高把位、難弓法。給程度高的演奏者 / 想要華麗版的人。
一句旋律 幾乎每個和聲點都加雙音
