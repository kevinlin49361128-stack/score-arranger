視覺特效 — 規劃決策

勾選想加進 app 的特效。多數卡片有「▸ 試看」可直接預覽效果(深色框模擬 app 內)。

選完按右下角「複製決策摘要」貼回給我。標 🎯 的是我的推薦,但你說了算。

判斷原則:受眾是音樂老師與學生 —— 多半非技術背景。對他們,特效不該是 「炫」,而該是「讓人安心、看得懂、覺得這軟體可靠」。所以最好的特效有一個 共同點:它同時是功能回饋。純裝飾的特效對這個受眾反而扣分。

因此這份清單刻意排除:
  • 彩帶 / 粒子 / 慶祝爆破 —— 對專業音樂教育者是「降格」訊號。
  • UI 音效 —— 這是音樂 app,介面叮咚會跟真正的樂音打架。
  • 全域重漸層 / 玻璃擬態 —— 退流行快,且譜面需要一塊安靜的視野。
  • 關鍵路徑上的長動畫 —— 揭示動畫一律 ≤400ms 且不阻塞操作。
全部點子皆為純 CSS / React,不碰引擎,低風險。

2 特效即回饋 可複選

這一類的特效同時是「功能回饋」—— 最推薦,因為它幫使用者看懂 app 做了什麼。

變動小節閃光 🎯 最推薦
套用加難度 / 降難度 / 抹平 / 移調後,被改動的小節短暫 泛起柔光。老師按「降難度」就看得到 AI 動了哪幾小節。可複用既有 issue→譜面的 flash highlight 程式碼。

價值 極高   工程量 低
播放游標質感化

套用成功微回饋
套用編輯後按鈕一個輕脈動 / 浮現勾號,不用 toast 那麼重。
價值 中   工程量 低

3 整體質感 可複選

不直接「做事」,但讓 app 整體像「正經、被細心打磨過的軟體」。

對話框動態 🎯 建議
改譜 / 難度調節等對話框現在是「彈」出來。改成 ~150ms 淡入 + 輕微放大、背景遮罩同步淡入。最便宜的全 app 質感升級。
