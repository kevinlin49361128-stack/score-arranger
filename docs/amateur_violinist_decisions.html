業餘小提琴手友善化

Score Arranger 改善決策文件 · 2026-05-28 · 基於 0.1.53 程式碼 recon
Persona: 30-50 歲成人, Suzuki book 3-5, 練 1-3 hr/週, 目標「明年婚禮拉 Pachelbel Canon」或「教會復活節獨奏 Ave Maria」。 改編家工具的功能他們用不上, 真正卡關的是: 找不到想拉的曲、降難度不會降把位、印出來沒指法、練習工具不齊。

改善方案 — 勾選你想做的

P0 M · 4-6h A. 曲庫業餘友善化 — 加 10 首 PD 名曲 🎯 建議

加 8-12 首業餘小提琴手熱門 PD 古典曲目, 並在 Onboarding 加「熱門曲」入口。

候選曲目

Pachelbel Canon in D · Massenet Méditation from Thaïs · Bach Air on G String · Vivaldi Spring/Winter 1st mvt · Schubert Ave Maria · Elgar Salut d'Amour · Mendelssohn Wedding March · Bach Cello Suite 1 Prelude (transposed for vln) · Bach Minuet in G · Boccherini Minuet

反方
Kevin 鎖定教學/古典市場, 但這些 本身就是古典 PD, 不是流行樂, 反而是「古典入門包」, 與品牌一致。風險: 部分曲目需從 IMSLP 抓 .xml 並補 metadata, 耗時。
P0 M · 6-10h B. 真正的 amateur 降難度 — 限把位 + 簡化節奏 🎯 建議

擴 simplify.py + violin profile 加: amateur skill → 偏好 1-3 把位 (用 fingering DP cost 加把位懲罰), 三十二分音符合併成十六分, 雙音保持單音 (已有)。

技術做法

在 core/simplify.py 加 simplify_amateur_violin(score): 用既有 viterbi fingering DP 強化 position transition cost, 高音音符優先用 E 弦低 fret 而非 G 弦高 fret 演奏; 額外把超出 3 把位的音 transpose 8va 下 (檢查可彈)。

反方
Vivaldi a-minor 中段轉 5 把位是必要的, 強制 1st position 會丟音樂表現。折衷: 啟發式 偏好低把位而非絕對 cap, 留 user 在 settings 開「嚴格 1st only」開關。
P0 S · 3-5h C. PDF 帶上指法 + 把位記號 🎯 建議

讓 IR 已算好的 viterbi 指法 / 把位真的印到 PDF 上。

技術做法

musicxml_writer.py 序列化 NoteEvent 時加 <notations><technical><fingering>3</fingering><string>2</string></technical></notations>; verovio 自動 render 指法數字 + 弦號。

弓法 (up/down bow) 留 Phase 2
Bow direction 自動推導不準, 加上去可能誤導使用者。先只做 fingering + string, bow 留 user 自填或下一階段做。
P1 S · 1-2h D. metronome 整合

練習基本工具。PlaybackControls.tsx 加 metronome toggle, Tone.js 用 woodblock sample 跟著 tempo 打點; 練習面板有獨立按鈕。

P1 M · 6-8h E. 練習日誌 + 進度追蹤

zustand 加 practiceLog slice: {date, score_id, duration_min, last_measure, score?, notes}。練習面板 sidebar 顯示「今日 / 本週累計」+ 曲目練習次數。麥克風練習評估自動寫入。

反方
練習日誌偏離「改編」核心, 像在做另一個 app。但 amateur 真實 journey 是「改編 → 練習 → 看進步」, 整合反而 lock-in。
P1 M-L · 8-16h F. 小提琴音色升級 (vibrato + dynamic layer)

找 CC0 violin sample (Sonatina Symphonic / Versilian) 加 vibrato layer + 多 dynamic; 長音自動 vibrato, 快音自動直音。

反方
Binary 已 199M, sample 加完可能 400M+。折衷: 走 CDN 動態載入, fallback 現有 Tone.Sampler — 但 packaged Electron 離線需求衝突。
P2 S · 2-3h G. Amateur Onboarding 分支

Onboarding 加第 0 步:「你是 ?」(改編家 / 業餘演奏者 / 老師) → 走不同 flow。業餘 flow 用 A 方案的 PD 名曲當範例, 預設 skill_level=amateur。

P2 L · 16h+ H. 即時麥克風跟拍 (現在只給「彈完評分」)

MicPracticePanel.tsx 升級成「即時 follow」: 拉到哪一個小節, OSMD cursor 自動同步; 拉錯顏色提示。技術: pitch detection per frame + 隱馬可夫對齊到譜面 onset 序列。

反方
Pitch tracking + score following 是博士論文等級題目, ROI 低於 D/E/F。但做出來是「殺手級」差異化, 可放長期 backlog。

建議組合

速勝包 (P0 三項, 約 13-21 hr, 拆 2 個 sprint): A + B + C — 把 amateur 客群實際拉得了的曲目、改得下的難度、印得清楚的譜全部做齊, 是「業餘小提琴手友善」最核心的三塊。

完整包 (P0+P1): 加 D + E, 多 ~10 hr, 練習工具補齊。

長期 (P2): G 改 onboarding, F/H 等真有人抱怨再做。
