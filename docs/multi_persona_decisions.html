業餘小提琴 / 中提琴 / 大提琴 / 低音大提琴 / 鋼琴 / 大鍵琴 / 木管 / 銅管 / 指揮 — 0.1.54 基線
核心發現: 0.1.54 把小提琴族打到可用級, 其餘 7 個 persona 各有不同程度的缺口。最嚴重的是移調樂器整條鏈路斷 — Clarinet/Trumpet/Horn 玩家拿到的譜記成實音, 根本不能用; 低音大提琴 first-class 支援度 0% (沒 profile / 沒 clef / 沒 catalog)。中提琴有 viola 譜號但無 onboarding 入口; 大提琴 36 樂章 Bach Suite 只有 1 首; 鋼琴左右手切分純 Middle C 太粗; 指揮工具完成度其實 60-70% 但缺最後一哩。

🔴 Critical Bugs / 阻斷性 P0-bug

P0 L · 12-20h A. 移調樂器整條鏈路修復 🎯 建議

Clarinet B♭ / Trumpet B♭ / Horn F / Saxophone 玩家拿到的譜一律記實音 — 譜上寫 C 實際 sounding B♭, 學的指法跟譜上完全對不上。IR 已有 Pitch.written_midi / written_spelling 欄位但 parser 沒填、writer 不輸出 <transpose> 元素, profile 雖標 transposition 完全沒讀。

技術做法

parser.py 從 music21 atSoundingPitch 計算寫入 written_midi/written_spelling; ir_to_musicxml.py 加 <transpose><diatonic/><chromatic/>; TargetPanel.tsx 給 toggle: 顯示記譜音 vs 實際音。

木管銅管指揮
反方
木管/銅管 amateur 用戶數比小提琴少, 但這是「拿到譜根本不能用」的 hard block; 修一次解 4-5 種樂器, ROI 不低。
P0 M · 8-12h B. 低音大提琴 first-class 支援 🎯 建議

整個 persona 在 app 內支援度 0%: 無 instruments/double_bass.py, 無 Bass8vbClef 自動套, 無 transcribe 慣例移調, ensemble template 列表 0 hit, catalog 0 首 walking bass / jazz。

技術做法

新建 double_bass.py profile (4 弦 E1/A1/D2/G2, range_amateur 1-3 把位); ir_to_music21.py 加 Bass8vb 自動套; transcriber.py 補 cello↔double_bass 慣例; arrangement_model.py 加 string_quintet (含 db)。

低音大提琴
反方
低音大提琴業餘人口最少 (估 <5%), ROI 客觀低於其他 P0。但「0% → first-class」是一次性投入, 後續無維護成本, 收尾感強。
P0 L · 20-30h C. 多樂章單檔 IR 支援

parser.py:288 寫死 score.movements = [Movement(...)], Mozart K458 / Haydn Op.74 必須拆 4 個檔案。指揮想 walkthrough 整曲、設置 tempo timeline、A/B 比較跨樂章, 通通不能做。

技術做法

IR 加真正 movement 概念; parser 偵測 multi-rest / double-bar 切分樂章; UI 加樂章 tab navigator。風險: 影響面廣 (改編 / 修復 / 匯出 / 播放都要適配)。

指揮所有演奏者
