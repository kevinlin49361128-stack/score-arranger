3 託管方案 — 網頁放哪

都是靜態頁,以下全部免費、零維運。差別在設定方式。

GitHub Pages 🎯 建議
  • 免費、和 repo 綁一起,push 即部署
  • .dmg 直接放同 repo 的 Releases,下載連結最自然
  • 前提:repo 要放上 GitHub
Cloudflare Pages
  • 免費、CDN 快、自訂網域容易
  • 多一個 Cloudflare 帳號 / 設定
Netlify
  • 免費、拖檔即上線、可加表單
  • 免費額度足夠,但又一個平台
自有網域 + 上述任一
  • scorearranger.com 之類,最正式
  • 網域年費(約 NT$300–500/年),要續約

4 贊助方案 — 收款管道

都需要你自己開帳號(我不能代辦)。台灣收款狀況我標在下面。

Ko-fi 🎯 建議
  • 一次性贊助平台不抽成(僅金流手續費)
  • 單次 / 訂閱皆可,介面友善、國際通用
  • 收款走 PayPal / Stripe(台灣可)
Buy Me a Coffee
  • 介面親切、好上手
  • 平台抽 5%
GitHub Sponsors
  • 平台 0 抽成,和開源最搭
  • 需 GitHub 帳號;台灣個人收款資格需先確認
綠界 ECPay / PayPal.me
  • 台灣在地金流,台灣使用者熟悉
  • 綠界偏商用設定較繁;PayPal.me 最陽春
先不放贊助
  • 純免費、零金錢元素,最無負擔
  • 之後想加隨時能加

5 頁面語言

app 本身已有中/英 i18n。網頁要不要雙語影響觸及範圍。

中文為主
先服務中文使用者,簡單。
中英雙語 🎯 建議
音樂教育市場以英文為大宗,雙語觸及更廣。

6 要不要開源 app

純屬選項。開源能換社群信任與貢獻;不開源也完全沒問題。

開源
公開原始碼,社群可貢獻,專案不會因你沒空而消失。
閉源但免費
只發佈成品 app,原始碼不公開。
之後再決定
不擋網頁進度,這題晚點再想。
